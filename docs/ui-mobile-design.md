# EnglishPod 365 — 移动优先 UI 设计规范（初稿）

> 目标：以移动端为核心，形成统一的视觉与交互语言；兼容桌面浏览器，同时为后续迭代（付费订阅、更多课程模块）预留空间。

---

## 1. 设计原则
- **Mobile First**：以 375–430 px 宽度为基础稿，桌面端通过扩展留白与双栏布局适配。
- **Consistency**：统一色板、字体、阴影、圆角，组件复用。
- **Legibility**：正文字号 ≥ 16 px，文本对比度 ≥ 4.5。
- **Accessibility**：触控目标 ≥ 44 px；交互反馈清晰（加载、错误、成功）。
- **Scalability**：组件化设计，方便扩展订阅、更多教学功能。

---

## 2. 品牌基调
| 类型 | 色值 | 用法 |
|------|------|------|
| 主色 Primary | `#0EA5E9` | CTA、主按钮、选中状态 |
| 强调 Accent | `#14B8A6` | 子 CTA、成功状态 |
| 错误 Error | `#F87171` | 表单校验、错误提示 |
| 背景 Base | `#F3F4F6` | 页面背景、卡片分隔 |
| 文字 Text | `#111827`（主）、`#6B7280`（次） | 正文、提示 |

- **字体**：移动端 `system-ui`，桌面端可加载 `Inter`（500/600）；中文使用苹方/思源黑体。
- **圆角**：卡片、按钮统一 `12px`；底部浮层顶部圆角 `16px`。
- **阴影**：`0 8px 24px rgba(15,23,42,0.08)`；浮层 `0 -6px 18px rgba(15,23,42,0.12)`。
- **间距**：竖向 16 / 24 / 32 px（小 / 中 / 大）；横向 16 px；组件内部 14 px。

---

## 3. 底部导航（移动）
- 结构：`课程 | 复习 | 我`，居中 Icon + Label，当前 Tab 使用主色填充。
- 音频播放时，底部导航上方保留迷你音频条（见组件）。
- 桌面端顶部导航：左侧 Logo，中间 Tab，右侧用户头像。

## 4. 核心页面布局

### 4.1 课程列表（Courses）

```
┌────────────────────────────┐
│ Header: Logo · Avatar      │
├────────────────────────────┤
│ Banner Card (渐变 + CTA)   │
│  "继续学习" Button         │
│  今日进度/连续天数         │
├────────────────────────────┤
│ 今日任务卡 (继续学习/复习) │
├────────────────────────────┤
│ 推荐课程瀑布列表 (卡片)     │
│  - 封面图 / 难度 / 时长     │
│  - 收藏按钮 / 进入课程      │
├────────────────────────────┤
│ 订阅预留区域（静态）       │
└────────────────────────────┘
```

- 桌面端：Banner + 推荐课程左列；右列显示任务摘要、订阅预留。

### 4.2 课程详情（Course Detail）

```
Header: ← 返回  | 标题 | ☆ 收藏
Hero: 课程封面 + 播放按钮 + Tag
Highlights Chips: 关键短语横向滚动
Tabs (吸顶): 概览 | 字幕 | 词汇 | 练习 | 播客
  - 概览: 简介、学习目标（列表）、讲师提示卡
  - 字幕: 按段落折叠，支持单句播放
  - 词汇: 卡片（词、释义、例句、收藏）
  - 练习: 上次得分、练习说明、进入按钮
  - 播客: 播客音频卡 + 文稿
Sticky Bottom Bar:
  - 左：迷你音频条 (标题 + 进度 + 播放键)
  - 右：主按钮“开始练习”
```

### 4.3 课后练习（Practice）

```
Header: ← 返回 | 课后练习 | 历史
Top Info Card: 上次提交时间、得分、重做按钮
Tabs: 完形填空 | 作文
 完形：
   Passage + 下拉空格（点击弹出 BottomSheet 选择）
   底部主按钮「提交完形填空」
   成绩卡 (得分 + 每题解析折叠)
 作文：
   提示卡（题目 + 要求）
   多行输入框 (自动扩展)
   字数统计 (实时)
   底部主按钮「提交作文」
   成绩卡 (总体反馈 / 问题讲解 / 修正范文)
历史记录 BottomSheet:
   列表：日期 | 完形得分 | 提交耗时 | 查看详情按钮
```

### 4.4 复习页（Review）

```
Header: 复习
Stats Card: 今日任务、完成数、连续天数
闪卡：
   全屏卡片（词汇/释义）
   顶部按钮：TTS 播放 + 释义开关（眼睛 Icon）
   底部按钮：记住了 / 还没掌握
   支持左右滑动切换
列表：
   搜索栏
   列表卡片（词汇、释义、例句、星标）
推荐卡片：继续巩固 XX 课程
```

### 4.5 用户中心（Me）

```
Profile Card：头像、昵称、学习天数、积分
学习报告：最近课程进度条、练习得分折线图
功能入口：
   我的收藏 / 练习记录 / 词汇本 / 设置
静态“订阅权益”卡片（按钮：敬请期待）
退出登录按钮
离线缓存卡片：展示缓存课程数量 + 管理按钮
```

### 4.6 登录 / 注册
- 渐变背景 + 品牌文案
- 右侧表单卡片：
  - 顶部 Toggle：登录 / 注册
  - 表单字段：用户名、（注册）邮箱、密码
  - 验证码区域：登录失败后露出
  - CTA：登录或注册按钮

### 4.7 订阅介绍（静态）
- 顶部 Banner：“高级会员即将上线”，按钮“加入候补名单（敬请期待）”。
- 权益卡片（课程拓展、AI 练习加强、离线缓存等，角标 `SOON`）。
- 价格计划占位表（无实际购买流程）。

## 6. 关键组件说明

| 组件 | 说明 |
|------|------|
| `BottomNav` | 移动底部导航，支持迷你音频条叠加。 |
| `StickyAudioBar` | 显示音频标题、进度条、播放按钮；点击展开全屏播放器。 |
| `Tabs` | 手势左右滑动；吸顶时带阴影。 |
| `Card` | 标准信息块，支持横向/纵向。 |
| `Chip` | 标签、难度、关键短语；可带图标。 |
| `BottomSheet` | 下拉选项、历史记录查看。 |
| `Flashcard` | 复习页核心组件，支持滑动手势。 |
| `Toast` | 底部中间显示 2.5s。 |

## 7. 响应式规则
- `min-width: 768px`：页面宽度限制 `max-width: 960px`，使用两栏布局。
- `min-width: 1280px`：考量增加边栏（课程推荐/练习快捷入口）。
- 音频条、底部按钮在桌面端改为固定在内容右侧面板。

---

## 8. 设计交付与开发计划
1. **高保真稿**：基于本规范在 Figma 绘制 6 个核心页面（移动/桌面），供审阅。
2. **组件库实现**：自 `apps/web-next` 建立 `components/ui`，按 Token 设计 Button / Card / Tabs / BottomNav 等。
3. **页面重构顺序**：
   - 课程列表（含底部导航 + Banner + 推荐卡片）
   - 课程详情（Hero + Tabs + Sticky 底栏）
   - 课后练习（完形 + 作文）
   - 复习页（闪卡 + 列表）
   - 用户中心 / 订阅介绍
4. **测试**：在 iOS Safari、Android Chrome、Chrome/Edge 桌面进行手动测试；使用 Lighthouse 评估性能与可访问性。
